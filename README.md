# ⚗ Little Alchemist Deck Optimizer

A browser-based deck optimizer for *Little Alchemist*. Build and manage your card library, then let the optimizer find the highest-scoring deck from your collection — entirely client-side, no server required.

**[Live demo →](https://vladajankovic.github.io/Little-Alchemist-Deck-Optimizer/)**

---

## Features

- **Library management** — add, edit, and remove cards with level, fused, onyx, and quantity attributes
- **Deck builder** — manually compose a deck or auto-fill it using the optimizer
- **Three optimizer modes**
  - ⚡ **Fill Deck** — greedy one-pass fill from your library
  - 🧠 **Advanced Fill** — iterative swap-based improvement
  - 🔁 **Try All Cards** — exhaustive search over every possible start card
- **Suggestions panel** — live ranked list of the best next card to add
- **Combination Matrix** — visual breakdown of all active combos in the deck
- **Leaderboard** — save, compare, import/export top deck builds
- **Light / dark theme** toggle
- **Persistent state** — library, deck, leaderboard, and settings survive page reloads via `localStorage`

## Setup

### Requirements

- Python 3.9+ with [openpyxl](https://pypi.org/project/openpyxl/)
- Node.js 18+

```
pip install openpyxl
```

### First-time setup

1. Place your `LA_vX-XX-XX_X.xlsm` file in the project root.

2. Parse the Excel file to produce `app/combo_data.json`:

   ```
   python app/data_loader.py
   ```

   Or pass the path explicitly:

   ```
   python app/data_loader.py path/to/your_file.xlsm
   ```

3. Install Node dependencies (first time only):

   ```
   npm install
   ```

4. Start the dev server:

   ```
   npm run dev
   ```

   Then open the URL shown in the terminal (e.g. `http://localhost:5173`).

### Production build

```
npm run build
```

Outputs a fully optimised, self-contained site to `dist/`. The contents of `dist/` are automatically deployed to GitHub Pages on every push to `master` via the included GitHub Actions workflow.

### Rebuilding after a game update

Re-run `python app/data_loader.py` whenever you have a new `.xlsm`, then commit the updated `combo_data.json`.

## Scoring Modes

The optimizer supports four scoring modes (configurable via ⚙ Settings):

| Mode | Formula |
|------|---------|
| 1 – Sum | `BA + BD + lvSum × cmbRare × 2` |
| 2 – Attack | `BA + lvSum × cmbRare × 2` |
| 3 – Defence | `BD + lvSum × cmbRare × 2` |
| 4 – Heroics | `BA×AB + BD×DB + lvSum × cmbRare × (AB+DB)` |

Where `lvSum = avgLevel + resAdj`, `cmbRare` is the combination rarity class, and `AB`/`DB` are the configurable Heroic Mode buffs.

---

## Settings

Accessible via ⚙ Settings in the top bar:

| Setting | Description | Default |
|---------|-------------|---------|
| Mode | Scoring formula (1–4) | 1 |
| Lowest Combo Worth Counting | Minimum score for a combo to count | 35 |
| Step Value | Score increment per combo step | 1.0 |
| Copy Reduction | Score multiplier for duplicate cards | 0.8 |
| Fusion Buff | Score multiplier for fused cards | 2.0 |
| Heroic Attack Buff | Attack multiplier in Heroics mode | 1.5 |
| Heroic Defence Buff | Defence multiplier in Heroics mode | 0.5 |
| Cards to Fill | Number of cards the auto-fill targets | 35 |
